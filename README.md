# InsightLearnAI

> Drop in a lecture, walk away with notes you can actually study from.

InsightLearnAI turns any video lecture into a complete study pack — full notes, a voice summary you can listen to on the commute, and a 10-question quiz — and then **measures how effectively the session actually worked** using a small ML model trained from scratch.

---

## What's in here

### The core study pipeline
1. **Audio extraction** from any uploaded video (FFmpeg).
2. **Speech-to-text** using OpenAI Whisper running locally — no transcription API costs.
3. **Comprehensive notes** generated by Llama 3.3 70B via Groq — proper study notes with definitions, examples and worked steps, not a thin summary.
4. **Voice summary** narrated by Microsoft Edge-TTS, with graceful fallback to the macOS `say` command if Edge throttles.
5. **10-question MCQ quiz** with explanations after every question.

### The learning-effectiveness layer (the part that's actually novel)
1. **Engagement tracking** — measures time you actively spend reading the notes (only while the tab is focused) and the real audio playback time (not just whether the file loaded).
2. **Quiz scoring** — captures correct / total and time taken.
3. **Custom-trained sentiment classifier** — TF-IDF + Logistic Regression on a hand-labeled dataset of 548 feedback sentences. **Test accuracy: 0.83 / macro-F1: 0.83.** Reads your free-text feedback and predicts a 1–10 score.
4. **Composite session grade** — combines the three axes into a single A/B/C/D grade with a visual breakdown:
   ```
   overall = 0.30 · sentiment  +  0.30 · quiz  +  0.40 · engagement
   ```
5. **Append-only logs** of every feedback row and every session in `backend/data/`, so you can chart trends later.

### Built-in Demo Mode
A **"Take a quick tour"** button on the home page loads canned notes, audio and a 10-question quiz so anyone can experience the full app without burning a Groq API call.

## Getting started

### 0. Prerequisites
- macOS or Linux (Windows works with adjustments)
- **Node.js 18+** for the frontend
- **Python 3.9+** for the backend
- **FFmpeg** on your PATH — `brew install ffmpeg` on macOS
- A **free Groq API key** from <https://console.groq.com/keys>

### 1. Clone and install the frontend
```bash
git clone https://github.com/sahils38/InsightLearnAI.git
cd InsightLearnAI
npm install
```

### 2. Backend setup
```bash
cd backend
python3 -m venv venv
source venv/bin/activate

# Install most deps the easy way…
python -m pip install -r requirements.txt

# …openai-whisper needs a small workaround on Python 3.9 + modern pip:
python -m pip install setuptools wheel
python -m pip install --no-build-isolation openai-whisper==20231117
```

### 3. Configure the Groq API key
```bash
cp .env.example .env
# open .env and replace the placeholder with your real key
# GROQ_API_KEY=gsk_…
```

### 4. Run it
Terminal 1 (backend):
```bash
cd backend
source venv/bin/activate
python main.py
# → Uvicorn running on http://0.0.0.0:8000
```

Terminal 2 (frontend) — from the project root:
```bash
npm run dev
# → http://localhost:8080  (note: not 5173)
```

Open <http://localhost:8080>, drop a video, or click **"Take a quick tour"** to try Demo Mode without uploading anything.

## API

| Method | Path | What it does |
|---|---|---|
| `POST` | `/api/upload` | Upload a video; returns `task_id` |
| `GET` | `/api/status/{task_id}` | Pipeline progress + `failed_step` + `error` on failure |
| `GET` | `/api/results/{task_id}` | Notes, quiz, audio URL |
| `GET` | `/api/audio/{task_id}` | Stream the TTS summary |
| `DELETE` | `/api/task/{task_id}` | Clean up files |
| `POST` | `/api/feedback` | Score one piece of text feedback with the ML model |
| `GET` | `/api/feedback/stats` | Aggregate stats over all logged feedback |
| `POST` | `/api/session-report` | Comprehensive: text + engagement seconds + quiz → composite grade |

Full schemas at <http://localhost:8000/docs> (Swagger UI).

## The ML model in 60 seconds

```
text → clean (lowercase, strip punct, drop stopwords)
     → TF-IDF vectorizer (1–2 grams, 3000 features, sublinear_tf)
     → LogisticRegression (class_weight="balanced")
     → {positive, neutral, negative} + class probabilities
     → score = round(((P(pos) − P(neg) + 1) / 2) × 10)   ∈ [1, 10]
```

- Dataset: **548 hand-labeled sentences** in `backend/ml/dataset.csv` (197 negative, 180 neutral, 171 positive).
- Test split: 80 / 20, stratified.
- **Test accuracy: 0.83. Macro-F1: 0.83. Per-class precision 0.78–0.86.**
- Retrain anytime with `python -m ml.train_model` from `backend/`.

Why this stack? Small hand-labeled dataset, interpretable coefficients, sub-millisecond inference, no GPU needed. We can swap in DistilBERT later — the data and training loop are ours.

## Resilience details worth knowing

- **TTS fallback chain** — Edge-TTS (2 retries) → macOS `say` + ffmpeg → silent placeholder. The pipeline never dies on a Microsoft 403.
- **Per-step error reporting** — the upload pipeline tags every failure with its step name. The UI shows the raw exception plus a "Likely cause / Suggested fix" hint that pattern-matches against the 8 most common failure modes (Groq key, whisper missing, ffmpeg missing, 403, 429, etc.).
- **Demo Mode** — bypasses Groq, Whisper and Edge-TTS entirely. Useful for offline demos.
- **Active-time tracking** — the `useTimeOnTab` hook stops counting when the page is hidden or the window loses focus, so engagement numbers reflect real attention.
