# Viva Demo Script — InsightLearnAI

A click-by-click run sheet for the live demo. **Read this once before walking in.** Print it or keep it on your phone.

---

## 0. Pre-flight (do this 10 minutes before)

Run through this checklist; abort and use Demo Mode (Section B) if any step fails.

```bash
# Terminal 1 — backend
cd /Users/<you>/Downloads/InsightLearnAI-main/backend
source venv/bin/activate
python main.py        # expect: "Uvicorn running on http://0.0.0.0:8000"

# Terminal 2 — frontend (from project root)
cd /Users/<you>/Downloads/InsightLearnAI-main
npm run dev           # expect: "Local: http://localhost:8080"

# Terminal 3 — quick smoke tests
curl http://localhost:8000/health
# expect: {"status":"healthy"}

curl -s -X POST http://localhost:8000/api/feedback \
  -H "Content-Type: application/json" \
  -d '{"text":"the notes were really clear"}' | head -c 200
# expect: {"label":"positive","score":...}
```

Open `http://localhost:8080` in Chrome. **DO NOT** open a Lovable preview URL — that returns `"No auth data provided"` errors.

Have a short test video ready (under 30 seconds, under 50 MB) on your desktop. Or commit to Demo Mode.

## A. The "real upload" demo path (when everything works)

Use this if your Groq key, internet, and Edge-TTS are all cooperating.

| Step | What you click | What you say |
|---|---|---|
| 1 | Drag a short video onto the uploader | "The pipeline has five steps. You'll see them tick by." |
| 2 | Point at the progress bar showing steps 1→5 | "Step 1 extracts audio with ffmpeg, step 2 transcribes locally using Whisper, step 3 calls Groq with Llama 3.3 70B for the notes, step 4 narrates them with Edge-TTS, step 5 generates a 10-question quiz." |
| 3 | When results load, scroll through the **Notes** tab | "Notes are full study notes, not just a summary — we explicitly prompt for definitions, examples and worked steps." |
| 4 | Switch to **Voice Notes** tab and press play | "Voice version is ~90 seconds, generated by Microsoft Edge-TTS. If their endpoint 403s, we fall back to the macOS `say` command — the pipeline never dies on a TTS hiccup." |
| 5 | Switch to **Quiz** tab — answer 3 or 4 questions, half correct half wrong | "Quiz has 10 questions. We track both correctness and time taken — both feed into the session report." |
| 6 | Click "See Results" on the quiz, point at the score + time | "Notice the time-taken line — that becomes one of our engagement signals." |
| 7 | Scroll down to **"How did this session go?"** | "This is where the project gets interesting. It combines three things into one composite grade." |
| 8 | Point at the strip: Reading / Listening / Quiz | "Reading time comes from a hook that only counts when the tab is visible and focused. Listening time comes from real audio playback events — pausing or seeking stops the counter. Quiz comes from your actual quiz attempt." |
| 9 | Type 1–2 sentences of feedback, click **"Show me the report"** | "The free-text feedback gets scored by an ML model we trained from scratch — I'll explain that in a second." |
| 10 | Point at the four gauges + the giant A/B/C/D grade | "30% sentiment from the ML model, 30% quiz score, 40% engagement. We give a single A/B/C/D grade so a student can see if they actually learned something or just scrolled through." |

## C. The ML model deep-dive (when they ask "tell me about the ML")

Open `http://localhost:8000/docs` in a new tab. Show **POST /api/feedback** in Swagger. Then say:

> "The feedback model is a sentiment classifier I built from scratch — no pretrained transformer.
>
> The pipeline is **TF-IDF with 1–2 grams and 3000 features**, feeding a **Logistic Regression** with balanced class weights.
>
> Training data is **548 hand-labeled feedback sentences** I wrote — 197 negative, 180 neutral, 171 positive.
>
> **Test accuracy is 0.83, macro-F1 is 0.83, per-class precision between 0.78 and 0.86.** That's on an 80/20 stratified split.
>
> Why this stack and not a transformer? Three reasons: small hand-labeled dataset, interpretable coefficients we can inspect, and sub-millisecond inference with no GPU. For production we'd swap in DistilBERT — but the data and training loop are ours."

Then live-demo it: in Swagger, **Try it out** → POST `/api/feedback`:
```json
{ "text": "honestly the quiz was too easy and the notes felt thin" }
```
Point at the response — `label: "negative"`, score around 3, with probabilities for all three classes.

## D. Q&A prep — answers to likely questions

| If they ask… | Say… |
|---|---|
| **Why three classes, not five-star?** | "Cleaner training data, fewer ambiguous boundaries. The 1–10 score is derived from class probabilities — `round(((P(pos) − P(neg) + 1) / 2) × 10)` — so we still get fine-grained output." |
| **How does the score map to 1–10?** | Show the formula above. "It produces a smooth 1–10 even when the predicted label is neutral. Pure-class output would lose that nuance." |
| **What if the lecture is in another language?** | "Whisper detects 99 languages. The Groq prompt asks for output in a target language, defaulting to English." |
| **How do you stop the audio counter being gamed by leaving the tab open?** | "We attach to the audio element's `play`/`pause`/`seeking`/`ended` events. Time only accumulates between actual `play` and `pause` events, not wall-clock time." |
| **What's the worst part of the project?** | Pick honesty over polish: "The dataset is hand-labeled by me, so the model inherits my labeling style. With 5x more data it'd handle sarcasm and mixed-tone feedback better." |
| **What would you do next?** | "Bigger dataset, character n-grams to catch typos, swap LogReg for DistilBERT fine-tuned on the same data, then active learning — surface low-confidence predictions for re-labeling." |
| **Why two separate ML scoring formulas (engagement vs sentiment)?** | "Engagement is a heuristic — capped time-on-task. Sentiment is ML. We keep them separate so the model isn't blamed for things it can't observe." |
| **Where's the data stored?** | "Append-only CSVs in `backend/data/`. Production would move to SQLite or Postgres, but a CSV is honest about what the demo is doing." |

## E. If something breaks live

| Symptom | What to do |
|---|---|
| "Failed to fetch" | Backend isn't running. Calmly say "let me restart this" and `Ctrl+C` → `python main.py`. |
| Stuck at step 2 (transcribing) for >2 min | Whisper is downloading the model (~140 MB) on first use. Say "first time loading the model, it caches afterward." |
| Step 3 fails with Auth/401 | Groq key issue. Switch to Demo Mode (path B) and explain the resilience: "We have a fallback demo mode that doesn't need the cloud API." |
| Step 4b fails with 403 | Edge-TTS throttling — but the macOS say fallback should auto-kick. If it doesn't, the silent placeholder still finishes the pipeline. Point at the "TTS fallback chain" line in the README. |
| Anything else | Open `/docs` Swagger and demo the ML model directly. The custom model is the examinable artifact — even if the orchestration breaks, the ML model itself is rock-solid. |
